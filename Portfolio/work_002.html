😥 購買流程不順利

我們從 GA 觀察到 user 在購買流程中停留比較久，甚至是直接關閉離開，因此想觀察是什麼原因讓他們需要花比我們預期還久的時間、並了解是什麼原因讓他們終止行為。

😥 收聽不便與需求

收聽會是使用者最常用的功能
我們想了解播放面板操作上是否有不直覺、資訊不明確或操作不流暢的問題產生？
競品所提供的功能，我們的使用者是否有期待功能或強需求，ex.倍速播放、定時關閉、待播清單。

😥 首頁缺乏吸引力

使用者停留在首頁時間平均不到30秒，且點擊率偏低，點擊率偏低代表停止使用的機率較高，因此進而影響了購買的意願。

挑戰

🎯 優化購買與加值流程

金流交易為非常重要的一項流程，除了系統機制要夠安全，user 在使用上，資訊必須要能一目瞭然明確，在支付流程時也要能感到放心為我們的首要目標。

🎯 優化收聽功能

收聽為這個產品蠻重要的行為，基本的元件有置底的小面板與滿版的播放面板。我們希望滿足 user 的需求來進行規劃與後續的調整。

🎯 確保資訊層次分明、功能導向明確

為了讓首頁帶給既有或新的使用者更有吸引力的設計，進而提高課程點擊率，將原本以列表形式排版的首頁進行了一次小改版。
增加新元件後，是否會影響使用者對於首頁的觀感？該新增哪種形式的元件是這次研究的目的。

1Goals

  1. 個人背景訪談
    1. 了解使用者的使用環境、歷程、裝置是什麼
  2. 易用性測試
    1. 了解使用者如何進行購買、加值、使用優惠的流程

    1. 了解使用者對於測試時的使用體驗

  4. 功能卡片排序
    1. 請使用者從9張卡片清單選出三個願望功能，為什麼會選擇這三項功能？

2測試回饋

2-1 購買流程

Learning 1

😥 購買流程，未提供使用者足夠的資訊

  1. 移除 Underlay 滿版廣告，調整內頁廣告的外觀和位置與數量，讓廣告與新聞內容區分明顯，且不過於干擾閱讀流程。
  2. 加值頁面使用者常忽略錢包剩餘餘額，而加錯金額。

Learning 2

😥 購買驗證的系統狀態顯示不明

  1. 加值頁面跳出 IAP 介面後，背景仍有Loading 畫面，使用者不知道該不該等到消失再確認購買。
  2. 購買頁面使用者按下立即兌換按鈕時，兌換時間有時會因為網路問題造成延遲，使用者不知道是否有成功點擊按鈕。

Learning 3

😥 優惠券輸入時的障礙

  1. 紅包頁面使用者進行輸入時，無法確定大小寫有沒有差異。
  2. 使用者覺得手寫輸入序號較麻煩，希望提供QR code輸入。

2-2 收聽功能

Learning 1

😥 播放面板違反的易用性原則

  1. 迷你播放面板「關閉」不易點擊，容易誤觸到開啟面板區域。
  2. 使用者想拖拉迷你播放面板的播放進度條。

Learning 2

😥 大面板介面辨識度低

大面板進度條辨識度低，使用者找不到哪裡可以控制音頻的時間進度。

Learning 3

😥 路徑迷失，找不到路徑

  1. 紅包頁面使用者進行輸入時，無法確定大小寫有沒有差異。
  2. 使用者覺得手寫輸入序號較麻煩，希望提供QR code輸入。

2-3 首頁

Learning 1

😥 缺乏推薦引導

降低了使用者瀏覽的動力。

Learning 2

😥 資訊層次不夠分明

導覽的視覺區隔性、功能識別性不足。

Learning 3

😥 產品吸引力不足

產品內容及功能不夠豐富，缺少回購的動力。

解決方案

1購買流程

重組購買頁階層並調整資訊欄位

1-1 欄位資訊顯示與微調

購買頁與加值頁，強化資訊欄位的識別性，降低使用者購買流程上的不安。

1-2 顯示流程狀態

使用者按下立即兌換按鈕後，加上狀態提示，以告知使用者已有按下按鈕並在進行兌換中。

2收聽功能

大面板、小面板 GUI 調整

2-1 大面板 UI 調整、新增功能

解決功能的能見度，並提供使用者的功能需求。

2-2 小面板 UI 優化、錯誤解決

解決使用者對於 UI 認知上的誤會，並解決操作上的錯誤。

3首頁改版

希望藉由改版來提高使用者參與度與黏著度

3-1 加強引導

提高使用者持續性的瀏覽動力與參與度，降低使用被中斷的機率。

3-2 新增首頁元件

  1. 希望藉由新增元件以提升整體視覺多樣性，增加使用者在首頁停留時間且提升點擊率。
  2. 不同元件的呈現方式，也解決了首頁屬性區隔性不夠的問題 code輸入。

4首頁改版設計實驗

4-1 橫向滑動與非橫向滑動顯示元件對於使用者在首頁觀感上的差異？

若首頁充滿大量橫向滑動區塊是否會影響使用者對於首頁的觀感？因此設計上，我們規劃了A/B兩種版本設計方案以進行比較，兩個版本差異如上。

☺️ 營收取得成長

在商業面來看，營收成長最快的就是在首頁的改版之後，因新的元件讓點閱率在一個月成長了接近 62%，進而大大影響到了使用者希望了解更多內容的動力。

成果與反思

透過這個專案中各種不同的合作方法，學到了如何從競品研究和市場觀察中找到產品的優勢，如何透過內部討論和使用者調查了解使用者需求並提出改善建議。

這個專案讓我也學習到如何從不同的觀點來設計產品，包括使用者和營運者的觀點。你學會了如何設計吸引人且能留住使用者的產品，以及最終如何促使客戶進行購買動作。最終，我們不僅提供了更好的使用者體驗，還實現了多贏的成果。

這些技能和知識對我未來在產品設計和營運方面的發展非常有幫助。這些經驗也證明了有效的合作方法可以為產品的成功和使用者的滿意度做出重要貢獻。
